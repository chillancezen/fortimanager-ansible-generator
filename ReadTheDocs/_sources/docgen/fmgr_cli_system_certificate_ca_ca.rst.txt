fmgr_cli_system_certificate_ca_ca -- CA certificate.
++++++++++++++++++++++++++++++++++++++++++++++++++++

.. versionadded:: 2.10

.. contents::
   :local:
   :depth: 1


Synopsis
--------

- This module is able to configure a FortiManager device by allowing the user to **[delete, get, set, update]** the following FortiManager json-rpc urls.
- `/cli/global/system/certificate/ca/{ca}`
- Examples include all parameters and values need to be adjusted to data sources before usage.
- Tested with FortiManager v6.0.0


Requirements
------------
The below requirements are needed on the host that executes this module.

- ansible>=2.10.0

   - The module may supports multiple method, every method has different parameters definition

Examples
--------

.. code-block:: yaml+jinja

 - hosts: fortimanager-inventory
   connection: httpapi
   vars:
      ansible_httpapi_use_ssl: True
      ansible_httpapi_validate_certs: False
      ansible_httpapi_port: 443
   tasks:
    - name: send request to /cli/system/certificate/ca/{ca}
      fmgr_cli_system_certificate_ca_ca:
         method: <value in [set, update]>
         url_params:
            ca: <value of string>
         params:
            - 
               data: 
                  ca: 
                   - <value of string>
                  comment: <value of string>
                  name: <value of string>
